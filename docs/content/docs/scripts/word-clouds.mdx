## Modules

- `scripts/word_clouds.py` ŌĆö orchestration, file I/O, palette selection (~1585 lines)
- `scripts/word_cloud_renderers.py` ŌĆö pure-SVG rendering backends with four layout strategies (~1231 lines)

<Callout type="warn" title="Two config models">
`WordCloudSettingsModel` in `config.py` is the top-level YAML config (part of `ProjectConfig`). `WordCloudSettings` in `word_clouds.py` is the internal generator config with `extra="forbid"`. Extra keys silently pass in the first, raise `ValidationError` in the second.
</Callout>

## What it generates

SVG word clouds where:
- **Topics cloud** ŌĆö keywords from repositories you've starred, sorted by topic frequency
- **Languages cloud** ŌĆö programming languages across your starred repos

Each cloud uses a hand-tuned OKLCH color palette and is output as an SVG (via matplotlib ŌåÆ SVG backend).

## Input data

The word cloud reads a markdown file produced by the `starred` CLI tool:

```
.github/assets/topics.md    ŌåÉ topic-grouped starred repos
.github/assets/languages.md ŌåÉ language-grouped starred repos
```

These are updated nightly by the `update-starred-lists` workflow job.

## CLI

```bash
# Topics word cloud
uv run python -m scripts.cli generate word-cloud \
  --config-path ./config.yaml \
  --from-topics-md \
  --output-path .github/assets/img/wordcloud_wordle_by_topics.svg

# Languages word cloud
uv run python -m scripts.cli generate word-cloud \
  --config-path ./config.yaml \
  --from-languages-md \
  --output-path .github/assets/img/wordcloud_wordle_by_languages.svg
```

## Configuration

Under `word_cloud_settings` in `config.yaml`:

```yaml
word_cloud_settings:
  output_dir: .github/assets/img
  output_filename: word_cloud.png
  prompt: "My Tech Skills: Python, JavaScript, Cloud, AI, DevOps, SQL, React"
  stopwords: []         # Additional stopwords list
```

## Output files

| File | Description |
|---|---|
| `wordcloud_wordle_by_topics.svg` | Topics from starred repos |
| `wordcloud_wordle_by_languages.svg` | Languages from starred repos |

## Rendering backends

`word_cloud_renderers.py` provides four pure-SVG layout strategies with OKLCH color palettes:

| Renderer | Strategy |
|---|---|
| `WordleRenderer` | Classic Wordle-style random placement |
| `ClusteredRenderer` | Grouped by semantic clusters |
| `TypographicRenderer` | Typography-focused layout |
| `ShapedRenderer` | Mask-based shape filling |
