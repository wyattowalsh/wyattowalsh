## Why modes exist

The living-art suite writes different artifacts for different viewers because the richest SVG output is not the most portable output. GitHub README embeds, social previews, image viewers, and workflow artifact inspection all place different constraints on what survives.

## Output matrix

| Surface | Canonical artifact | Compatibility artifact | Why |
|---|---|---|---|
| Ink Garden | `inkgarden-growth-animated.svg` | `inkgarden-growth.gif` | Stacked SVG is richer, GIF survives inline embedding |
| Topography | `topo-growth-animated.svg` | `topo-growth.gif` | Static/cartographic fallback is easier to read in flattened viewers |
| Genetic Landscape | `genetic-growth-animated.svg` | `genetic-growth.gif` | Evolutionary terrain benefits from both high-fidelity SVG and portable GIF |
| Physarum | `physarum-growth-animated.svg` | `physarum-growth.gif` | Network growth is best seen animated; GIF gives broad compatibility |
| Lenia | `lenia-growth-animated.svg` | `lenia-growth.gif` | Continuous automata need smooth playback; GIF provides universal fallback |
| Ferrofluid | `ferrofluid-growth-animated.svg` | `ferrofluid-growth.gif` | Metallic spike dynamics are richest in SVG; GIF for inline embedding |
| Timelapse exports | `living-*.gif` | n/a | Timelapse is already flattened for portability |

## CLI modes

### `generate living-art`

Runs the shared six-style `scripts.art.animate` driver twice by default:

1. `--svg --frames <svg-frames>` to write stacked timeline SVGs
2. `--frames <frames> --size <size>` to write compatibility GIFs

Use `--svg-only` when you want the richer source artifacts without rewriting the compatibility GIFs.
Use `--only inkgarden`, `--only topo`, `--only genetic`, `--only physarum`, `--only lenia`, or `--only ferrofluid` when you want to regenerate just one style.

```bash
uv run readme generate living-art \
  --metrics-path /tmp/metrics.json \
  --history-path /tmp/history.json \
  --only topo \
  --svg-only
```

### `generate timelapse`

Produces daily-history GIFs for one or all styles. These are export-oriented artifacts rather than source SVGs.

```bash
uv run readme generate timelapse \
  --metrics-path /tmp/metrics.json \
  --history-path /tmp/history.json \
  --only inkgarden \
  --max-frames 120 \
  --size 400
```

## Constraints to document and test

| Constraint | Consequence |
|---|---|
| Timeline SVGs are source artifacts, not guaranteed inline playback assets | Link to them, do not rely on them for universal animation support |
| Topography and Ink Garden can render static-safe frames | Frame generation should disable timeline CSS when producing GIF inputs |
| Timelapse outputs are intentionally flattened | They are review/export artifacts, not interactive sources |

<Callout type="idea" title="Recommended release review">
Inspect both tiers: the source SVG for richness and the GIF preview for actual README behavior. A beautiful SVG that collapses into an unreadable flattened preview is still a release blocker.
</Callout>
