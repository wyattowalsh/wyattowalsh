---
title: Word Clouds
description: OKLCH-palettized word clouds generated from your GitHub starred-repo topics and languages.
---

## Modules

- `scripts/word_clouds.py` — orchestration, file I/O, palette selection
- `scripts/word_cloud_renderers.py` — rendering backends (matplotlib/wordcloud integration)

## What it generates

SVG word clouds where:
- **Topics cloud** — keywords from repositories you've starred, sorted by topic frequency
- **Languages cloud** — programming languages across your starred repos

Each cloud uses a hand-tuned OKLCH color palette and is output as an SVG (via matplotlib → SVG backend).

## Input data

The word cloud reads a markdown file produced by the `starred` CLI tool:

```
.github/assets/topics.md    ← topic-grouped starred repos
.github/assets/languages.md ← language-grouped starred repos
```

These are updated nightly by the `update-starred-lists` workflow job.

## CLI

```bash
# Topics word cloud
uv run python -m scripts.cli generate word_cloud \
  --config-path config.yaml \
  --techs-path .github/assets/topics.md \
  --output-path .github/assets/img/wordcloud_by_topic.svg

# Languages word cloud
uv run python -m scripts.cli generate word_cloud \
  --config-path config.yaml \
  --techs-path .github/assets/languages.md \
  --output-path .github/assets/img/wordcloud_by_language.svg
```

## Configuration

Under `word_cloud_settings` in `config.yaml`:

```yaml
word_cloud_settings:
  output_dir: .github/assets/img
  output_filename: word_cloud.png
  prompt: null          # Optional LLM prompt for keyword curation
  stopwords: null       # Additional stopwords list
```

## Output files

| File | Description |
|---|---|
| `wordcloud_by_topic.svg` | Topics from starred repos |
| `wordcloud_by_language.svg` | Languages from starred repos |
