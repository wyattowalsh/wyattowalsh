## 1. Core Scripts (`[@scripts/](mdc:scripts)`) Detailed Breakdown

### 1.1. `[@scripts/banner.py](mdc:scripts/banner.py)`

- **Purpose:** Generates a highly customizable, visually rich SVG banner.
- **Key Libraries:** `svgwrite`, `numpy`, `pydantic`, `loguru`, `noise`.
- **Configuration (`pydantic` Models):**
  - `BannerConfig`: Main config (dimensions, title, subtitle, colors, typography, effects, layout, generative patterns, output settings).
  - `ColorPalette`: Multiple color schemes, dark mode support.
  - `Typography`: Fonts, sizes, weights, spacing, text effects (shadow, gradient, glow), responsive adjustments.
  - `VisualEffects`: Blur, glassmorphism, Perlin noise, glow, shadows, chromatic aberration, grain.
- **Inputs:** `BannerConfig` model instance. Reads `[@.github/assets/img/octocat.svg](mdc:.github/assets/img/octocat.svg)`.
- **Outputs:** SVG file (e.g., `[@.github/assets/img/banner.svg](mdc:.github/assets/img/banner.svg)`).
- **Key Functionality:** SVG drawing, filter/gradient definitions, background rendering, complex visual pattern generation (chaotic attractors, flow fields, neural nets), Octocat embedding, text rendering with dynamic contrast adjustment, optional `svgo` optimization.
- **Invocation:** `make generate-banner`, `python scripts/cli.py generate banner`, `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` (Job: `generate-assets`).

### 1.2. `[@scripts/qr.py](mdc:scripts/qr.py)`

- **Purpose:** Generates a vCard QR code PNG with an artistic SVG background.
- **Key Libraries:** `segno`.
- **Configuration:** vCard data from `config.yaml`, background path `[@.github/assets/img/icon.svg](mdc:.github/assets/img/icon.svg)`, default output `[@.github/assets/img/qr.png](mdc:.github/assets/img/qr.png)`, default scale.
- **Inputs:** vCard data from `config.yaml`, static `[@.github/assets/img/icon.svg](mdc:.github/assets/img/icon.svg)`.
- **Outputs:** PNG file (e.g., `[@.github/assets/img/qr.png](mdc:.github/assets/img/qr.png)`).
- **Key Functionality:** Structures vCard, generates QR object (error correction 'H'), renders artistic PNG with SVG background.
- **Invocation:** `make generate-qr`, `python scripts/cli.py generate qr_code`, `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` (Job: `generate-assets`).

### 1.3. `[@scripts/word_clouds.py](mdc:scripts/word_clouds.py)`

- **Purpose:** Generates word cloud PNGs from topics/languages in Markdown files.
- **Key Libraries:** `wordcloud`, `markdown`, `beautifulsoup4`, `Pillow`, `numpy`, `matplotlib.colors`, `pydantic`.
- **Configuration:** Default input paths (`[@.github/assets/topics.md](mdc:.github/assets/topics.md)`, `[@.github/assets/languages.md](mdc:.github/assets/languages.md)`), default output paths (`[@.github/assets/img/wordcloud_by_topic.png](mdc:.github/assets/img/wordcloud_by_topic.png)`, etc.), `WordCloud` parameters from `config.yaml` or CLI.
- **Inputs:** Markdown files from `[@.github/assets/](mdc:.github/assets)` (dynamically generated by `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` Job: `update-starred-lists`).
- **Outputs:** `wordcloud_by_topic.png`, `wordcloud_by_language.png`.
- **Key Functionality:** Reads MD, converts to HTML, parses HTML for terms & frequencies (from `<a>` tags and `<li>` counts in `<ul>`), generates cloud from frequencies, saves to PNG.
- **Invocation:** `make generate-word-clouds`, `python scripts/cli.py generate word_cloud --techs-path <md_file>`, `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` (Job: `generate-assets`).

### 1.4. `[@scripts/cli.py](mdc:scripts/cli.py)`

- **Purpose:** `typer`-based CLI for script execution with parameters, using `config.yaml` for defaults.
- **Key Libraries:** `typer`.
- **Main Commands:**
  - `config`: Manage `config.yaml` (view, save, generate-default).
  - `generate <entity_type>`: (`banner`, `qr_code`, `word_cloud`).
    - `banner`: Wraps `[@scripts/banner.py](mdc:scripts/banner.py)`. Options for dimensions, output, optimization, title, subtitle override `config.yaml`.
    - `qr_code`: Wraps `[@scripts/qr.py](mdc:scripts/qr.py)`. Options for error correction, scale, background, output override `config.yaml`.
    - `word_cloud`: Wraps `[@scripts/word_clouds.py](mdc:scripts/word_clouds.py)`. `--techs-path` for input MD, `--output-path`, `--prompt` override `config.yaml`.
- **Note:** `[@Makefile](mdc:Makefile)` primarily uses this CLI for generation tasks. The `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` workflow also uses this CLI.

### 1.5. `[@scripts/techs.py](mdc:scripts/techs.py)`

- **Purpose:** Intended for generating a Markdown section of technologies. Currently parses tech data from Markdown using Pydantic models and can display it. Its data can be used by `[@scripts/cli.py](mdc:scripts/cli.py)` for generic tech word clouds.
- **Status:** Not a stub. Contains Pydantic models (`Technology`), `load_technologies` (MD parser), `display_technologies` (Rich output). A dedicated function for generating a structured Markdown output (e.g., a 'Technologies I Use' section for `[@README.md](mdc:README.md)`) and a direct CLI command for this specific output are key future enhancements.
- **Inputs (inferred):** Markdown file (e.g., `[@techs.md](mdc:techs.md)`, `[@.github/assets/topics.md](mdc:.github/assets/topics.md)`).
- **Outputs (intended):** Markdown file or console output.
- **Action:** Define and implement `generate_techs_markdown_section(input_path: Path, output_path: Path) -> str` and integrate with `[@scripts/cli.py](mdc:scripts/cli.py)` as a new subcommand (e.g., `python scripts/cli.py generate techs_section`). Consider if this script should also produce other outputs like categorized lists or data for other visualizations.

## 4. GitHub Actions (`[@.github/workflows/](mdc:.github/workflows)`) Overview

### 4.1. `[@.github/workflows/profile-updater.yml](mdc:.github/workflows/profile-updater.yml)` (Unified Profile Updater)

- **Purpose:** Consolidated workflow for all profile automation tasks.
- **Triggers:** `workflow_dispatch`, `schedule` (daily), `push` (master/main).
- **Key Jobs:**
  - `update-starred-lists`:
    - Uses `actions/setup-python@v5` (Python 3.13, `uv`).
    - Installs `.[script-tools]` (`starred` CLI).
    - Runs `starred` to generate/update `languages.md` and `topics.md` in `[@.github/assets/](mdc:.github/assets)`.
    - Commits changes using `stefanzweifel/git-auto-commit-action@v5`.
  - `generate-assets`:
    - Depends on `update-starred-lists`.
    - Uses `actions/setup-python@v5` (Python 3.13, `uv`).
    - Installs `.[qr,word-clouds,banner]`.
    - Runs `python scripts/cli.py generate qr_code`, `word_cloud` (for topics & languages), `banner` using `config.yaml`.
    - Commits generated assets in `[@.github/assets/img/](mdc:.github/assets/img)` using `stefanzweifel/git-auto-commit-action@v5`.
  - `update-readme-wakatime`:
    - Uses `anmol098/waka-readme-stats@v4` to embed WakaTime stats into `[@README.md](mdc:README.md)`.
  - `update-readme-blog`:
    - Uses `gautamkrishnar/blog-post-workflow@v1` to embed Medium blog posts into `[@README.md](mdc:README.md)`.
  - `generate-profile-metrics`:
    - Uses `lowlighter/metrics@latest` to generate profile metrics SVGs (`metrics.svg`, `metrics.additional.svg`) in `[@.github/assets/img/](mdc:.github/assets/img)`.
- **Permissions:** `contents: write` for jobs that commit changes.
- **Status:** Primary active workflow.
